# Stage 3 Guideline — Detail 输出

## 前置条件

- `final-analysis.md` 一~四章节已填充
- Basic + Core P0 缺口已补完

---

## 执行步骤

### Step 3.1: 填充五~八章节

读取 `assets/analysis-template.md`，基于已有信息 + 协作补充内容，填充五~八章节。

**特别注意：第五章指标**

若 PRD 中未提及"设计过程指标"，AI 需结合需求类型主动推测 1-2 个可量化设计指标并直接写入：

- 搜索类需求 → 推断"搜索成功率"、"平均搜索时长"
- 填表类需求 → 推断"表单填写完成时长"、"步骤放弃率"
- 流失挽回类需求 → 推断"挽回成功率"、"二次访问率"

**特别注意：第六章 MVP**

在列出需求表格前，基于"资源有限"的假设，主动提炼一段 MVP 战略分析：

- 哪些是保障核心价值跑通的最少功能集？
- 哪些看似美好但耗费开发资源的边缘场景应当被果断砍掉或延期？

**注意**：本阶段输出五~八章节（Detail）。九章节为 P2，不输出。

---

### Step 3.2: 输出模板参考

**五、业务目标与可量化指标**

所有指标必须可量化、可验证，无基线值、无目标值的模糊表述不填写。

- 5.1 业务北极星指标
- 5.2 可量化设计过程指标
- 5.3 体验兜底指标

**六、需求清单与优先级排序**

- 6.1 MVP 最小体验闭环定义
- 6.2 需求排序模型（RICE/KANO/四象限）
- 6.3 需求全清单与优先级分级

**七、核心落地策略**

- 7.1-7.4 产品策略、运营策略、设计策略

**八、风险预判与应对方案**

- 风险类型、风险等级、应对方案、兜底措施

## 门禁检查

### P0 缺口检查

检查以下 P0 缺口（见 `references/p0-gates.md`）：

**Detail 阶段 P0**：
- 5.1 业务北极星指标（指标名称 + 目标值）
- 6.3 需求全清单（至少列出 P0 需求）

有 P0 缺口 → 进入协作讨论环节
无 P0 缺口 → 进入协作确认环节（确认推断项），确认后创建 quality-report.md

## 协作讨论环节

通用流程见 `references/collaboration-protocol.md` §通用协作讨论环节流程
