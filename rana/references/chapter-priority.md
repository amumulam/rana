# 章节优先级定义 — P0/P1/P2

参照 `ref/需求分析与设计目标说明书模板.md` 的章节结构，定义各小节的缺口优先级。

---

## 分级原则

| 优先级 | 定义 | 影响 |
|-------|------|------|
| **P0** | 缺失会导致文档无法交付设计阶段 | 阻塞进入下一阶段 |
| **P1** | 缺失会影响设计决策质量 | 可带病推进，协作阶段补充 |
| **P2** | 缺失不影响核心交付 | 后续迭代补充，不阻塞 |

---

## Basic 阶段（一、二章节）

**P0 小节：**
- 1.1 文档基础信息：需求名称、所属业务线/产品、迭代版本号
- 2.1 需求来源：至少一项溯源依据（定量/定性/战略）
- 2.3 影响范围：覆盖用户群体

**P1 小节：**
- 1.2 版本迭代记录
- 2.2 历史复盘

---

## Core 阶段（三、四章节）

**P0 小节：**
- 3.1 现状描述：至少一个核心现象
- 4.1 核心用户画像：基础属性 + 痛点与诉求
- 4.3 核心使用场景：用户 + 场景 + 任务 + 障碍 四要素

**P1 小节：**
- 3.2 根因拆解
- 3.3 问题分级
- 4.2 次要用户画像
- 4.4 次要/边缘场景

---

## Detail 阶段（五~八章节）

**P0 小节：**
- 5.1 业务北极星指标：指标名称 + 目标值
- 6.2 需求全清单：至少列出 P0 需求

**P1 小节：**
- 5.2 可量化设计过程指标
- 5.3 体验兜底指标
- 6.1 需求排序模型
- 7.2 产品策略
- 7.3 运营策略
- 7.4 设计策略
- 8.0 风险预判表格

---

## P2 小节（不阻塞）

- 九、各角色重点关注问题（v0.3.0 不输出）
