# 可追溯性标注规范

> **说明**：此文件仅供内部质量验证参考，不在 skill instructions 中展示。用户可见的报告不强制展示来源追溯标记。

需求分析的每一项内容都必须标注来源。这是质量门禁的基础要求，也是后续设计决策可追溯的保障。

---

## 来源类型与标注格式

| 来源类型 | 标注格式 | 示例 | 说明 |
|---------|---------|------|------|
| PRD 章节 | `[PRD第X节]` | `[PRD第3节]` | 指向具体章节编号 |
| PRD 章节+段落 | `[PRD第X节第Y段]` | `[PRD第3.2节第1段]` | 需要精确定位时使用 |
| 截图 | `[截图X]` | `[截图1]` | 按提供顺序编号 |
| Figma 页面 | `[Figma页面: 名称]` | `[Figma页面: 登录流程]` | 用页面名称，不用编号 |
| PM 口头确认 | `[PM确认]` | `[PM确认]` | 对话确认的内容 |
| 研发确认 | `[研发确认]` | `[研发确认]` | 技术可行性确认 |
| 测试确认 | `[测试确认]` | `[测试确认]` | 测试覆盖性确认 |
| 业务确认 | `[业务确认]` | `[业务确认]` | 业务方确认 |
| 设计师推断 | `[来源推断]` | `[截图1推断：截图显示移动端界面]` | 必须带推断依据，裸 `[推断]` 不合法 |
| 分析方法推断 | `[方法名推断：依据]` | `[场景还原推断：基于PRD第1节用户描述]` | 五问法/X-Y分析/场景还原得出的推断 |
| 信息缺失 | `[缺失]` | `[缺失]` | 进入待澄清清单 |

## 标注原则

### 1. 每项必标

分析文档中的每一条信息，无论是功能描述、边界条件还是用户场景，都必须有来源标注。
不允许出现没有标注的分析内容。

### 2. 推断必须带依据

推断内容不能凭空出现。格式：`[来源推断]`

**正确**：
> 用户可能在移动端使用此功能。`[截图1推断：截图显示移动端界面]`

**错误**：
> 用户可能在移动端使用此功能。`[推断]`（缺少推断依据）

**分析方法推断示例**：
- `[五问法推断：追问至第3层，真实需求是快速定位未读消息]`
- `[X-Y分析推断：PM 提案是消息中心（Y），真实问题是消息感知缺失（X）]`
- `[场景还原推断：基于PRD第1节「用户反映错过消息」的心境还原]`

### 3. 缺失必须进入待澄清清单

标注了 `[缺失]` 的内容，必须同步加入待澄清清单（交付物 D），并指定确认对象。

### 4. 来源冲突必须处理

当两个来源对同一内容描述不一致时：
- 在 change-log.md 记录冲突和解决方式
- 标注以确认来源为准：`[截图1，已与PRD第3节冲突，以PM确认为准]`

## 标注示例

### 单来源标注

```markdown
- 用户点击「提交」按钮后，系统显示成功提示。`[PRD第4.2节]`
- 提示文案为「提交成功」。`[截图3]`
- 按钮在加载状态下变为灰色不可点击。`[Figma页面: 表单页]`
```

### 多来源标注（互相印证）

```markdown
- 列表最多显示 50 条数据。`[PRD第5节]` `[研发确认]`
```

### 推断标注

```markdown
- 该功能主要在 Wi-Fi 环境下使用。`[截图2推断：截图显示大量图片加载]`
```

### 来源冲突标注

```markdown
- 按钮文案为「确认提交」。`[截图1：「确认」；PRD第3节：「提交」；以PM确认为准，统一用「确认提交」]`
```

### 缺失标注

```markdown
- 超时时间限制：`[缺失]` → 已加入待澄清清单 #3，需研发确认
```

## 在各阶段的应用

| 阶段 | 标注要求 |
|------|---------|
| Stage 1 输入解析 | 每条提取的信息必须标注原始来源 |
| Stage 2 缺口分析 | 识别到的缺口标注 `[缺失]`；推断的内容标注 `[来源推断]` 或 `[方法名推断：依据]` |
| Stage 3 协作对话 | 对话中确认的内容标注 `[PM确认]` / `[研发确认]` 等 |
| Stage 5 交付物 | 所有交付物内容必须有来源列，100% 覆盖 |

## 验证器识别的完整格式列表

验证器通过正则匹配来源标注。只要符合以下任意一种格式，该行即视为"已标注"。

| 来源类型 | 验证器可识别的格式 | 说明 |
|---------|-----------------|------|
| PRD 章节 | `[PRD第X节]` `[PRD 第X节]` `[PRD第X.Y节]` `[PRD 需求清单]` `[PRD 封面]` | `[PRD...]` 开头均可 |
| 截图 | `[截图1]` `[截图2附注]` | 必须含数字编号 |
| PDF | `[PDF第1页]` `[PDF截图1]` | |
| Figma | `[Figma页面: 登录流程]` `[Figma组件: Card]` | `[Figma...]` 开头均可 |
| 对话确认 | `[PM确认]` `[研发确认]` `[测试确认]` `[业务确认]` `[设计师确认]` `[用户确认]` `[用户确认 19:16]` | 带时间戳也可 |
| 推断 | `[推断：依据]` `[推断]` `[分析推断]` `[场景还原推断：依据]` `[五问法推断：依据]` `[X-Y分析推断：依据]` | |
| 信息缺失 | `[缺失]` `[缺失：原因]` | |
| 口头说明 | `[口头说明：内容]` | |
| 变更记录引用 | `[CHG-1]` `[CHG-12]` | |
| 原始输入 | `[原始输入：内容]` | |
| 系统生成 | `[分析创建]` | 版本号等系统自动填入内容 |

> **不在上述列表中的格式不会被识别**，等价于"未标注"，会被计为来源缺失。
> 例如：`[analysis infer]`、`[来自PRD]`、`[设计稿]` 均不被识别。
