Quick Mode · workflow-quick-mode-guideline.md
QUICK MODE — 快速了解需求、辩证理解、挖掘需求
Step 1：快速分析（四维度）
问题 → 目标 → 供给 → 指标
强制约束：100%复制模板、禁止纯摘要、必须推断、暴露数据缺失
使用模板: analysis-template-quick.md
⚠️ 暂停输出，等待用户回复
强制输出完四表格后必须停止！抛出1-2个探讨性问题
禁止第一轮对话直接输出提问清单
Step 2：深度分析与讨论
多轮对话 · 人在环中
主动抛出探讨性问题 → 用户反驳 → 5步批判反驳流程
协作规范: collaboration-protocol.md
Step 3：评审澄清提问清单
≥3 个犀利提问，每个含：提问内容 + 追问理由 + 关联维度
输出: quick-analysis.md（澄清提问清单章节）
Step 4：升级提示
"若需完整 Full Mode 分析说明书，请回复「执行 Full Mode」"
输出: quick-analysis.md + change-log.md
Full Mode · workflow-full-mode-guideline.md
FULL MODE — 完整分析说明书 · 三阶段渐进完善
三阶段概览
S1
诊断层 · 二~四章
找病因 + 按需协作 · stage-1-diagnosis.md
S2
方案层 · 五~八章
开药方 + 按需协作 · stage-2-solution.md
S3
提炼层 · 一章+总结
倒推提炼 + 按需协作 · stage-3-refine.md
每个 Stage 结束：四段式独立消息
消息块1引导语（各阶段自定义）
消息块2逐字全文输出本阶段章节（不省略、不截断）
消息块3P0 缺口清单 + 待确认推断项 → 进入协作确认
消息块4确认提示语（各阶段自定义）
定义通用结构见 collaboration-protocol.md §四段式独立消息通用结构
协作确认环节（每阶段结束）
无 P0 缺口 → 确认推断项和存疑点 → 进入下一阶段
有 P0 缺口 → 多轮协作讨论 → 补完后确认
通用流程: collaboration-protocol.md §通用协作讨论环节流程
质量验证（仅 Full Mode）
Stage 3 完成后运行 quality-validator.py
文件结构 / 来源追溯 / P0小节 / 章节完整 / 模糊表述
python3 rana/scripts/quality-validator.py <分析目录> [--score]
