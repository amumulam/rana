# 需求分析卡

> 本文件为需求分析卡的权威格式模板。AI 执行时参照此结构填写，`## 填写示例` 区块仅供人工参考，不输出至交付物。
>
> **多需求场景**：PRD 包含多个独立功能需求（功能级子需求）时，每个需求单独输出一张卡片，依次罗列，用 `---` 分隔。  
> 禁止将多个需求的字段值用分号拼接写入同一行。
>
> **字段名强制约束**：分组名和字段名必须严格使用本模板定义的名称，**禁止自定义或替换**。错误示例：将"基本信息"改为"业务目标"，将"需求内容"改为"功能描述"。
>
> **禁止使用的自定义分组名（反例）**：`业务目标`、`功能描述`、`约束条件`、`验收标准`、`异常处理`。这些都是字段名或非标准分组名，不得作为 `▌` 分组行使用。

---

## 卡片格式

每张卡片由以下两部分组成：

1. **标题行**：`### [需求名称]（[需求ID 或标识]）`  
2. **单一表格**：所有字段在一张表格内呈现，用分组行划分 section。分组行格式：第一列填 `**▌ 分组名**`，其余列留空。

```markdown
### [需求名称]（[需求ID]）

| 字段 | 内容 | 来源追溯 |
|------|------|---------|
| **▌ 基本信息** | | |
| 需求名称 | | |
| 版本/迭代号 | | |
| 需求来源 | | |
| 背景说明 | | |
| 业务目标 | | |
| **▌ 用户与场景** | | |
| 目标用户 | | |
| 使用场景 | | |
| 核心问题 | | |
| **▌ 需求内容** | | |
| 需求拆解 | 见下文「需求拆解清单」 | — |
| 业务规则 | | |
| 约束条件 | | |
| 优先级 | | |
| **▌ 风险与待确认** | | |
| 待澄清项 | 见下文「待澄清问题清单」 | — |
| 风险点 | | |
| **▌ 结论** | | |
| 需求结论 | 见下文「需求分析结论」 | — |
```
