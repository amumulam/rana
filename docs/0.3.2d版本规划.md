# Rana v0.3.2d - 现场修复记录

**触发场景**：实际运行中发现 Stage 1-2 输出呈现指令未被执行，缺口表格格式问题。

---

## 问题清单

### 问题 1: Stage 全量展示指令未被执行

**现象**：用户反馈 Stage 1 完成后只显示摘要，没有展示完整的一~四章节内容。

**根因**：原有指令表述不够强调、不够精准：
- 未明确「逐字全文」
- 未明确使用代码块包裹
- 未明确「不得省略、摘要、截断」

### 问题 2: 缺口表格格式问题

**现象**：
- 表头使用「章节」，应改为「小节」
- 第一列显示「一、前置基础」等带编号前缀，用户希望简化为「前置基础」
- 示例表格版本号仍在 P0 缺口（实际已改为 P1）

---

## 修复方案

### 修复 1: Stage 1-3 强制输出呈现

在每个 Stage 结束节点增加强制输出指令：

```markdown
**Stage N 输出呈现（强制要求）**：

Stage N 结束前必须静默读取 final-analysis.md 全量内容，并在回复中用 markdown 代码块包裹逐字输出给用户。

- 必须输出 final-analysis.md **逐字全文**，不得省略、摘要、截断
- 使用 ` ```markdown ` 代码块包裹
- 输出前静默读取文件内容

附上确认提示语。
```

### 修复 2: 缺口表格格式优化

- 表头：`章节` → `小节`
- 第一列：去掉「一、」等编号前缀，简化为章节名
- 版本号示例：从 P0 缺口移至 P1 缺口
