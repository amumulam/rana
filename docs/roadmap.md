## 已发布版本

### v0.3.2b

**变更**：Stage 强制输出呈现、blockStreaming 三段式、移除来源标注格式说明。

### v0.3.2

**变更**：迁移成本评估（Stage 1）、HMW 方案发散引擎（Stage 2）、MVP 最小体验闭环（Stage 3）。

### v0.3.1

**变更**：咨询式批判思维人设、逻辑合理性诊断、启发式追问、5 步反驳机制、边界情况处理。

### v0.3.0

**变更**：输出结构从 5 交付物改为 9 章节说明书模板，引入四阶段渐进完善流程（Basic → Core → Detail → Refine）。

### v0.2.0

**变更**：知识库检索、交互细节判断标准。

### v0.1.0

**变更**：基础 5 交付物结构（A-E），Stage 0-5 流程定义。

## 规划中版本

### v0.3.3

**来源**: `docs/superpowers/specs/2026-04-15-v0.3.3-skill-refactor-design.md`

**定位**: SKILL.md 重构，拆解、精简、规范化。

| 功能 | 说明 |
|------|------|
| SKILL.md 精简 | 移除嵌入模板、Stage 详细步骤，精简至 200-300 行 |
| Stage guideline 拆分 | 创建 stage-1/2/3-guideline.md |
| P0 缺口规则汇总 | 创建 p0-gates.md |
| 协作对话规范提取 | 创建 collaboration-protocol.md |
| 文件命名规范化 | 全英文命名，归档文件 `_archived-` 前缀 |
| 模板迁移至 assets | 符合 skill-creat-guideline 规范 |
| 四段式输出统一 | 所有 Stage 统一四段式 + 缺口讨论环节 |

### v0.4.0

| 功能 | 说明 |
|------|------|
| 同需求多轮迭代关联 | 检测历史目录，继承/全新分析，变更标记 |
| 特殊需求类型适配 | BUG修复类/合规整改类/运营活动类/ToB定制类差异化流程 |

**需求类型适配**：

| 类型 | P0 调整 | 流程适配 |
|------|---------|---------|
| 功能迭代类（默认） | 标准 P0 | 标准 4 阶段 |
| BUG 修复类 | 用户画像/场景降为 P1 | Stage 1 仅一~三章节 |
| 合规整改类 | 合规红线强制 P0 | 增合规清单检查项 |
| 运营活动类 | 活动目标/指标为 P0 | Stage 3 侧重运营策略 |
| ToB 定制类 | 客户确认项为 P0 | 增客户确认环节 |

## 版本号规则

- **主版本号 (x.0.0)**: 核心流程重大变更
- **次版本号 (0.x.0)**: 新功能特性
- **修订号 (0.0.x)**: Bug 修复、文档更新、微调优化
