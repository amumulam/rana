# Rana 0.2.0 设计文档 — 文件预处理委托机制

**版本**: 0.2.0
**日期**: 2026-04-13
**状态**: Draft

---

## 1. 需求概述

### 核心需求

Rana 0.2.0 新增文件预处理能力，将 PDF/Word/PNG 等输入文件的解析**委托给外部 skill**，生成结构化 Markdown 后自动衔接 rana 的 Stage 1 分析流程。

### 关键设计要点

| 项目 | 决策 |
|------|------|
| 外部 skill 类型 | OpenClaw skill（内置 + 用户安装） |
| 配置文件位置 | skill 目录（`~/.openclaw/skills/rana/config.yaml`） |
| 配置文件格式 | YAML |
| 配置结构 | 按文件类型列出 skill 名称 |
| fallback 触发 | 配置不存在 + 对应类型未配置 |
| fallback 实现 | 模型多模态能力直接读取 |
| 流程衔接 | 自动衔接 Stage 1（无用户确认） |
| 子 agent 调用 | sessions_spawn + 每分钟轮询状态 |

## 2. 整体架构

### 工作流程变化

**Rana 0.1.0 流程**：

```
输入: PRD + 截图（文字/Markdown）
        │
        ▼
Stage 1: 输入结构化
        │
        ▼
Stage 2-6: 后续流程
```

**Rana 0.2.0 流程**：

```
输入: 任意文件（PDF/Word/PNG/等）
        │
        ▼
Stage 0: 文件预处理（新增）
        │
        ├─ 检测文件类型
        ├─ 读取 config.yaml，查找对应 skill
        ├─ 调用外部 skill（sessions_spawn）
        │   └─ 输出到指定目录
        │
        ▼
Stage 1: 输入结构化（自动衔接）
        │
        ▼
Stage 2-6: 后续流程不变
```

## 3. 配置文件设计

### 文件位置

`~/.openclaw/skills/rana/config.yaml`

### 配置结构

```yaml
# Rana 文件解析配置
# 指定各文件类型使用哪个 OpenClaw skill 解析
# 未配置的类型将使用 fallback（模型多模态解读）

file_parser:
  pdf: "mineru-pipeline"
  word: "mineru-pipeline"
  png: "mineru-pipeline"
  docx: "mineru-pipeline"
  # 用户可添加更多类型或替换为其他 skill
```

### 配置逻辑

```
检测文件类型（扩展名）
    │
    ▼
检查 config.yaml
    │
    ├─ 文件存在？
    │   ├─ YES → 查 file_parser.<类型>
    │   │   ├─ 找到？→ 调用指定 skill
    │   │   └─ 未找到？→ fallback
    │   └─ NO → fallback
```

## 4. Stage 0 详细流程

### 流程图

```
用户输入文件路径
        │
        ▼
    检测文件类型
    （提取扩展名：pdf/word/png/docx/等）
        │
        ▼
    读取 config.yaml
        │
        ├─ 找到对应 skill？
        │   ├─ YES → sessions_spawn 子 agent
        │   │   │
        │   │   ├─ 子 agent 加载指定 skill
        │   │   ├─ 执行文件解析任务
        │   │   ├─ 输出到 rana 指定目录
        │   │   │
        │   │   └─ 主 agent 每分钟轮询状态
        │   │       ├─ 正常进行 → 继续等待
        │   │       ├─ 完成 → 读取输出文件
        │   │       ├─ 异常 → 报告错误，让用户决定
        │   │
        │   └─ NO → fallback
        │       │
        │       ├─ 告知用户："未配置解析器，使用模型多模态解读"
        │       ├─ 调用模型多模态能力
        │       └─ 输出文字描述
        │
        ▼
    解析结果（Markdown 内容）
        │
        ▼
    自动衔接 Stage 1
```

## 5. sessions_spawn 子 agent 调用

### 子 agent 创建

**调用方式**：
```
sessions_spawn(
  task: "使用 {skill_name} skill 解析文件 {file_path}",
  agentId: "file-parser",
  skills: ["{skill_name}"]  # 注入指定 skill
)
```

### skill 注入

子 agent 配置 `skills: ["{skill_name}"]`，OpenClaw 自动加载该 skill 的 SKILL.md instructions 到子 agent prompt。

### 输出目录传递

调用 skill 时传递参数：
- `inputPath`: 用户文件路径
- `outputDir`: `ux-requirement-analysis/_temp/`

**mineru-pipeline 输出结构**：
```
ux-requirement-analysis/_temp/
└── {filename}/
    └── auto/
        ├── {filename}.md          ← 主 Markdown 文件
        ├── {filename}_images/     ← 图片
        └── ...
```

**读取输出**：主 agent 从 `{outputDir}/{filename}/auto/{filename}.md` 读取内容。

## 6. 状态轮询机制

### 轮询间隔

每 60 秒检查一次子 agent 状态。

### 状态判断

| 状态 | 行为 |
|------|------|
| `running`（正常进行） | 继续等待，下一轮再检查 |
| `completed` | 读取输出文件，进入 Stage 1 |
| `error` / `failed` | 报告错误，提供给用户选择（重试/fallback/手动输入） |

### 轮询方式

主 agent 与子 agent 通过 sessions 机制沟通，检查状态是否正常进行。

## 7. Fallback 机制

### 触发条件

1. `config.yaml` 文件不存在
2. 文件类型未在 `file_parser` 中配置

### Fallback 流程

```
检测到 fallback 条件
    │
    ▼
告知用户："当前文件类型未配置解析器，正在使用模型多模态能力直接解读"
    │
    ▼
调用模型多模态能力读取文件
（适用于图片、PDF 等模型支持的格式）
    │
    ▼
输出文字描述 → 临时 Markdown 内容
    │
    ▼
自动衔接 Stage 1
```

### 注意事项

- 不阻塞流程，告知用户后立即执行
- 输出格式与正常解析一致，供 Stage 1 使用
- 大文件可能耗时较长，但无超时限制

## 8. 错误处理

### 可能的错误情况

| 错误 | 原因 | 处理方式 |
|------|------|---------|
| skill 不存在 | config.yaml 中指定的 skill 未安装 | 报告错误，提示用户安装或修改配置 + 执行 fallback |
| 解析失败 | skill 执行出错（文件损坏/格式不支持） | 报告错误，让用户选择（重试/fallback/手动输入） |
| 输出文件缺失 | skill 完成但未生成预期输出 | 执行 fallback |

### 用户选择选项

```
解析失败。请选择：
1. 重试（再次调用 skill）
2. 使用 fallback（模型多模态解读）
3. 手动输入（您提供文字内容）
```

```markdown
## Stage 0: 文件预处理（新增）

当用户输入为文件（PDF/Word/PNG 等）而非纯文本时，rana 会先执行文件预处理：

1. 检测文件类型（扩展名）
2. 读取配置文件（`config.yaml`），查找对应 skill
3. 调用外部 skill 解析文件 → 输出结构化 Markdown
4. 自动衔接 Stage 1

**配置文件位置**：`~/.openclaw/skills/rana/config.yaml`

**Fallback**：若配置不存在或类型未配置，使用模型多模态能力直接解读。
```

```markdown

       │
       ▼
Stage 1: 输入解析与结构化    → input-structured.md
       │
       ▼
Stage 2-6: 后续流程不变
```

## 11. 验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | config.yaml 存在且格式正确 | `cat ~/.openclaw/skills/rana/config.yaml` |
| 2 | PDF 文件能被正确解析 | 输入 PDF → 生成 Markdown → 进入 Stage 1 |
| 3 | 未配置类型触发 fallback | 删除 config.yaml 或输入未配置类型 → 模型直接解读 |
| 4 | 子 agent 状态轮询工作 | 解析大文件时观察到状态变化 |
| 5 | 错误处理正确 | 指定不存在的 skill → 报告错误 + 提供选项 |
| 6 | Stage 1 自动衔接 | 解析完成后无需用户确认即进入 Stage 1 |
