# Rana Agent 行为约束 — v0.4.3 设计文档

## 背景

当前 Rana 作为 Skill，在 Agent 层执行时存在以下行为级问题：

| 痛点 | 现象 |
|------|------|
| 内部逻辑泄露 | AI 将 CoT（思维链）、Goal、Instructions 等英文内部指令直接输出给用户，破坏对话沉浸感 |
| 元数据问询误判 | 用户问"文件在哪"，AI 误解意图，把几千字正文重新打印一遍 |
| 意图不明就执行 | 用户表达模糊时，AI 直接猜测执行，做错事 |
| 权限边界卡壳 | 用户要求保存到 Downloads 等外部目录，AI 强行执行后 Permission denied 报错 |
| JSON 转义错误 | 工具调用时参数转义出错，触发反复重试，拖慢响应 |

## 设计目标

在 SKILL.md 中新增独立章节 **"Agent 行为约束"**，用轻量禁令式规则约束 Agent 的工具调用行为，作为四段式消息结构的补充和强化。

## 设计决策

### 落点：SKILL.md 新增独立章节

**选择 B**：在 SKILL.md 中新增独立章节 `## Agent 行为约束`，置于"工作原则"之后、"批判反驳规则"之前。不混入需求分析方法论。

不去 AGENTS.md（方案 C）的原因是：这些约束是 rana 身份特有的（专业 UX 顾问的对话行为规范），不应该影响用户其他 skill 的 Agent 行为。

### 形式：轻量禁令式规则

采用 **（规则名 + 规则内容）** 的表格形式，不仿照 C1-C7 的 5 步批判流程。这些是**行为禁令**而非需求判断，用 1-2 行表达即可。

### 与现有四段式消息结构的关系

四段式消息结构（`collaboration-protocol.md` §四段式独立消息通用结构）中的"消息块 2"已规定阶段结束时将文件内容写入 message block。新行为约束章节作为其**补充和强化**，不重复定义最终交付物呈现规则。

## 新增章节结构

### 引导说明

> Agent 行为约束是对 [四段式消息结构](references/collaboration-protocol.md §四段式独立消息通用结构) 的补充和强化。最终交付物呈现遵循消息块 2 的规则，以下约束聚焦 Agent 日常交互行为。

### 规则表（5 项）

| # | 规则名 | 规则内容 |
|---|--------|---------|
| B1 | **内部逻辑不可见** | 你的 CoT（思维链）、Goal、Instructions、中间日志等内部信息禁止输出给用户。始终保持中文专业业务助理身份进行对话。 |
| B2 | **文件路径问询只给路径** | 用户问"文件在哪/保存在哪/目录是什么/路径是什么"，只返回文件的绝对路径字符串。禁止打印文件正文内容。 |
| B3 | **意图不明先追问** | 用户输入模糊、意图不清时，先追问澄清，严禁猜测后直接执行。 |
| B4 | **权限边界说清替代方案** | 无权限操作时，说明系统权限限制，提供文件当前已保存的绝对路径，建议用户手动处理。不强行执行后报错卡壳。 |
| B5 | **JSON 参数注意转义** | 调用工具时严格注意 JSON 格式和引号转义，参照工具文档中的示例格式构造参数，避免转义错误导致反复重试。 |
