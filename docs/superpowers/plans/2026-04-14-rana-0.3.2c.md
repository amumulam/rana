# Rana v0.3.2b 修复计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复实际运行中发现的三个问题：1) Stage 1 全量展示指令不够明确；2) 示例缺口表格仍有版本号作为 P0；3) 表格格式优化（章节→小节，简化列值）。版本号保持 0.3.2b 不变。

**Architecture:** 修改 SKILL.md 的 2 个位置：Stage 1 输出展示指令加强、Stage 2 对话脚本示例表格。纯 prompt 修改。

**Tech Stack:** Markdown 文档编辑，无依赖变更

## 问题分析

### 问题 1: Stage 1 完成后没有全量展示

**现象**：用户反馈 Stage 1 完成后只显示摘要，没有展示完整的一~四章节内容。

**原因**：Stage 1 的全量展示指令在 SKILL.md 中已有，但可能：
- 指令不够强调（没有「必须」、不够醒目）
- 位置容易被跳过或忽略

**修复**：强化指令表述，使用醒目的 ⭐ 标记和「强制」要求。

### 问题 2: 示例缺口表格仍有版本号作为 P0

**现象**：Stage 2 的示例表格 `| 一、前置基础 | 版本号 | 请确认...` 依然显示版本号为 P0 缺口。

**原因**：虽然 P0 缺口清单已修改（移除版本号），但 Stage 2 对话脚本中的示例表格没有同步更新。

**修复**：更新示例表格，将版本号移至 P1 缺口示例。

### 问题 3: 缺口表格格式问题

**现象**：
- 表头用「章节」，应该用「小节」
- 第一列显示「一、前置基础」带括号，用户希望简化为「前置基础」

**修复**：
- 表头改为「小节」
- 示例表格第一列去掉「一、」，直接显示「前置基础」
