# v0.3.2b 修复计划

**日期**: 2026-04-14
**版本**: v0.3.2b（继续修复）
**来源**: 用户实际运行反馈

---

## 修复背景

用户在实际运行 rana agent 时发现两个问题：

1. **来源标注影响阅读体验**：报告中每个位置都展示 `[PRD第X节]`、`[截图X]`、`[用户确认]` 等溯源标记，影响文档可读性
2. **Stage 输出呈现代码块包裹问题**：markdown 代码块包裹会产生很多 markdown 字符，影响阅读；且每个阶段应该有针对性呈现（Stage 1 只输出一~四章）

---

## 修复项 1：移除来源标注格式说明

### 目标

从 SKILL.md 和参考文件中移除所有溯源标记说明，不再要求在报告每个位置展示来源追溯。

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| `rana/SKILL.md` | 移除 description 中的来源追溯说明、Stage 1 来源标注格式速查、底部的「来源标注规范」章节 |
| `rana/references/traceability-guide.md` | 保留文件但不再作为用户可见的参考，仅作内部质量验证参考 |

### 具体修改位置

#### SKILL.md

1. **description 部分**（第 11 行）
   - 移除：`每项分析标注来源追溯（[PRD第X节]、[截图X]、[分析推断]）。`

2. **Stage 1 输出模板前**（第 330-335 行）
   - 移除整段：
   ```
   **来源标注格式速查：**
   
   ```
   [PRD第X节] / [PRD X.X节]（如 PRD 1.2节）/ [截图X] / [PDF第X页] / [PDF截图X] / [用户确认] / [缺失] / [分析创建]
   ```
   ```

3. **Stage 3 第五章特别说明**（第 675-680 行）
   - 移除：`（标注 `[分析推断]`）`

4. **底部「来源标注规范」章节**（第 834-850 行）
   - 移除整个章节

5. **模板表格中「来源追溯」列**（多处）
   - 保留表格结构，但移除「来源追溯」列说明，改为可选或移除该列
